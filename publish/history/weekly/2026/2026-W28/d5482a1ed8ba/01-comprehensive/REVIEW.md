## 项目事实

### 01｜asgeirtj/system_prompts_leaks

- 仓库：https://github.com/asgeirtj/system_prompts_leaks
- 定位：汇集多家 AI 聊天产品、编程工具及历史版本的系统提示词，方便集中查阅和横向对照
- Star：56,147
- Fork：9,269
- 本期信号：+7,765 Star（trending）
- 许可证：CC0-1.0
- 能力：查阅不同厂商与产品的提示词；对照同一模型的不同版本；查看编程与办公助手的指令；定位工具定义和个性配置；从更新表找到新增提示词

### 02｜iOfficeAI/OfficeCLI

- 仓库：https://github.com/iOfficeAI/OfficeCLI
- 定位：通过单个命令行程序创建、读取和修改 Word、Excel、PowerPoint 文件，无需安装 Office
- Star：15,014
- Fork：1,022
- 本期信号：+5,789 Star（trending）
- 许可证：Apache-2.0
- 能力：创建并编辑常用 Office 文件；读取文本、样式、公式和图表；渲染文档并预览编辑结果；用 JSON 填充跨格式模板；导出结构并批量重建文档

### 03｜usestrix/strix

- 仓库：https://github.com/usestrix/strix
- 定位：让多个 AI 渗透测试 Agent 运行目标、验证漏洞，并交付复现证据、修复建议和报告
- Star：40,480
- Fork：4,256
- 本期信号：+6,443 Star（trending）
- 许可证：Apache-2.0
- 能力：扫描代码库、仓库或 Web 应用；验证漏洞并生成复现步骤；分析访问控制与注入问题；生成修复补丁和测试报告；在拉取请求中检查变更范围

### 04｜ogulcancelik/herdr

- 仓库：https://github.com/ogulcancelik/herdr
- 定位：在终端集中查看、拆分和恢复多个 Agent 会话，让任务离开当前终端后继续运行
- Star：15,460
- Fork：1,035
- 本期信号：+4,714 Star（trending）
- 许可证：AGPL-3.0-or-later
- 能力：查看多个 Agent 的终端输出；拆分窗格并用键鼠操作；分离会话后重新连接；通过套接字创建和读取窗格；安装插件扩展工作流

### 05｜openai/codex-plugin-cc

- 仓库：https://github.com/openai/codex-plugin-cc
- 定位：把本机 Codex 接入 Claude Code，用斜杠命令完成代码审查、任务委派和会话移交
- Star：27,615
- Fork：1,798
- 本期信号：+4,484 Star（trending）
- 许可证：Apache-2.0
- 能力：审查当前改动或分支差异；挑战设计假设和风险；委派排错与修复任务；查看、获取或取消后台任务；把当前对话移交给 Codex

### 06｜stablyai/orca

- 仓库：https://github.com/stablyai/orca
- 定位：把多种命令行编程 Agent 放进统一桌面工作区，用隔离 worktree 并行尝试、比较和合并结果
- Star：16,306
- Fork：1,274
- 本期信号：+4,328 Star（trending）
- 许可证：MIT
- 能力：并排运行多种编程 Agent；把同一任务分发到隔离 worktree；比较结果并合并选中实现；在差异行添加反馈；通过手机监控和引导任务

### 07｜diegosouzapw/OmniRoute

- 仓库：https://github.com/diegosouzapw/OmniRoute
- 定位：在本机建立统一 AI 接口，把编程工具请求转发到不同模型供应商，并在限额或故障时切换目标
