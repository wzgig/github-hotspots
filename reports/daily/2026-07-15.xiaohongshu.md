01｜skills
把软件工程方法做成可组合的代理技能，帮助开发者对齐需求、测试代码并控制架构复杂度

它能做什么：

- 追问需求细节并梳理待解决的决策分支
- 生成共享术语文档与架构决策记录
- 把讨论内容整理成规格或可执行工单
- 按红绿重构循环实现功能与修复缺陷
- 检查代码标准及其对原始规格的符合度

核心亮点：可拆装的工程工作流。每项技能只负责一类工程任务，可由用户主动调用或由代理按场景选用；从需求澄清、领域建模到实现和审查，开发者可组合流程而不必交出全部控制权。

适合谁：适合使用 Claude Code、Codex 等编码代理，并希望保留流程控制权的软件开发者
使用前准备：需可运行 npx，并在每个仓库先执行一次 /setup-matt-pocock-skills 完成配置。

使用提醒：插件安装得到只读托管包，不能直接编辑；Codex 原生插件仍在路线图中。

许可证：MIT

为什么本期上榜：过去 24 小时净增 +1,858 Star（历史快照核验）
GitHub 搜索：mattpocock/skills

02｜graphify
把代码、文档和媒体资料连成可查询的知识图谱，让编码代理沿关系查项目而不是逐文件搜索

它能做什么：

- 扫描项目并生成可交互的知识图谱
- 查询与问题相关的局部关系子图
- 追踪任意两个概念之间的连接路径
- 解释节点来源、关联关系与置信标记
- 合并代码、文档、PDF、图片及音视频信息

核心亮点：可追溯的项目关系图。代码通过 tree-sitter 在本地解析，关系边区分 EXTRACTED 与 INFERRED；生成的图谱可查询节点、路径和概念，并把代码与设计文档、媒体资料放进同一关系网络。

适合谁：适合需要理解陌生代码库、追踪模块关系或让编码代理查询项目上下文的开发者与团队
使用前准备：需要 Python 3.10+，并使用 uv 或 pipx 安装 PyPI 包 graphifyy。

使用提醒：代码可本地解析；文档、PDF 和图片的语义提取会调用助手模型或已配置的后端。

许可证：MIT

为什么本期上榜：过去 24 小时净增 +1,673 Star（历史快照核验）
GitHub 搜索：Graphify-Labs/graphify

03｜awesome-llm-apps
提供可克隆改造的 AI 代理、RAG 和代理技能样例，帮你从具体任务直接找到可运行起点

它能做什么：

- 克隆单文件代理并替换模型或业务逻辑
- 安装代理技能为编码助手增加专项任务能力
- 搭建多代理协作、路由与结果汇总流程
- 构建面向网页、PDF 和多媒体的 RAG 应用
- 参考可交互界面、语音代理和 MCP 集成样例

核心亮点：按任务挑选可运行样例。仓库把样例按入门代理、进阶代理、多代理团队、RAG、语音和生成式界面等场景组织；读者可先运行单个项目，再替换模型、数据源和工具形成自己的应用。

适合谁：适合想学习代理应用结构、验证产品想法，或从现成样例改造成业务工具的 Python 开发者
使用前准备：按所选样例准备模型 API Key，并安装对应目录的 requirements.txt 依赖。

许可证：Apache-2.0
