# 正文

2026-07-17 · D006 · AI 专题榜

AI 项目的名字往往很抽象。这一期不堆术语，只讲清它替谁做什么、能完成哪些任务，以及是否值得加入你的工具箱。

先看结论：
- hallmark：为 Claude Code、Cursor 和 Codex 生成不套模板的网页界面，也能审查或重做现有设计
- awesome-llm-apps：把智能体、RAG 和多模型应用整理成可克隆运行的示例，方便从具体任务开始改造
- hermes-agent：让个人 AI 助手跨会话记住经验、生成并改进技能，还能在终端或聊天平台持续执行任务

01｜hallmark
为 Claude Code、Cursor 和 Codex 生成不套模板的网页界面，也能审查或重做现有设计

它能做什么：

- 生成与需求匹配的网页界面
- 审查现有代码并输出问题清单
- 重做页面结构并保留原有内容
- 提取参考设计的结构与视觉特征
- 输出可交接给其他 AI 工具的 design.md

核心亮点：让不同需求长出不同页面结构。它先根据需求选择宏观结构和主题，再经过反套路检查与输出前自查；找不到合适主题时会从配色、字体到布局重新设计，而不是只替换模板颜色。

适合谁：适合用编码智能体制作网页、检查界面套路，或希望在保留内容的前提下重做视觉结构的人
使用前准备：需将技能安装到 Claude Code、Cursor 或 Codex；也可用 npx skills add nutlope/hallmark 安装。

使用提醒：设计研究不会生成像素级仿制品，也拒绝复制付费模板；审查模式只给问题清单，不会修改代码。

许可证：MIT

GitHub 搜索：Nutlope/hallmark

02｜awesome-llm-apps
把智能体、RAG 和多模型应用整理成可克隆运行的示例，方便从具体任务开始改造

它能做什么：

- 安装可供编码智能体使用的技能
- 克隆并运行单文件智能体示例
- 搭建多个智能体协作的任务流程
- 让模型检索私有资料后回答问题
- 连接浏览器、GitHub 与 Notion 等工具

核心亮点：按任务挑选可运行的 AI 应用。仓库不是单一框架，而是按技能、入门智能体、复杂智能体、智能体团队、语音、MCP 和 RAG 等场景组织代码；读者可先运行一个示例，再替换模型或改造任务。

适合谁：适合想用现成代码学习智能体与 RAG，或需要从模板改出研究、数据、内容和自动化工具的开发者
使用前准备：运行具体示例通常需克隆仓库、进入对应目录、安装 requirements.txt，并准备该示例所需的 API key。

许可证：Apache-2.0

GitHub 搜索：Shubhamsaboo/awesome-llm-apps

03｜hermes-agent
让个人 AI 助手跨会话记住经验、生成并改进技能，还能在终端或聊天平台持续执行任务

它能做什么：

- 切换模型提供方而无需修改代码
- 通过终端或聊天平台连续对话
- 搜索并总结过去的会话记录
- 创建子智能体并行处理不同工作流
- 按自然语言计划定时任务并推送结果

核心亮点：把经验变成会持续改进的技能。它把会话记忆、用户模型和技能系统连成学习闭环：复杂任务结束后可创建技能，使用过程中继续改进，并通过历史会话检索在后续任务中调用积累的经验。

适合谁：适合需要长期运行个人智能体、跨设备延续对话，或把重复工作交给定时任务和并行子智能体的人
使用前准备：安装后需选择模型提供方与模型；使用搜索、图像、语音或云端浏览器等工具时，还需配置对应服务。

使用提醒：手动克隆进行开发时，虚拟环境不能放在源码目录内；Termux 的完整依赖包含不兼容的语音组件。

许可证：MIT
