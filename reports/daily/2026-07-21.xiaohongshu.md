01｜code-review-graph
为大型代码库建立结构图，让 AI 审查变更时只读取相关文件与依赖

它能做什么：

- 解析函数、类、导入与调用关系
- 定位改动可能影响的文件和执行流
- 筛选审查任务所需的最小上下文
- 持续更新保存或提交后的代码变化
- 生成架构图、代码 Wiki 与多种导出文件

核心亮点：用代码图缩小审查范围。它用 Tree-sitter 解析仓库，把函数、类、导入和调用存成图；文件变化后沿依赖边寻找受影响代码，再把精简上下文交给 AI，而不是反复扫描整个项目。

适合谁：需要让 AI 审查大型仓库、分析改动影响或梳理跨文件依赖的开发者与研发团队
使用前准备：需要 Python 3.10+；安装并构建代码图后，应重启编辑器或 AI 工具。

使用提醒：小仓库和简单单文件改动可能得不偿失；影响分析会保守地多报文件，关键词排序和部分语言的执行流识别仍有不足。

许可证：MIT

02｜voicebox
把声音克隆、语音生成、全局听写和 AI 代理播报放进本地桌面工作流

它能做什么：

- 克隆参考录音或选用预设声音
- 生成多语言语音并添加音频效果
- 编辑多人对话、播客和叙事时间线
- 转写录音并整理口误与停顿
- 通过 REST API 或 MCP 接入应用与代理

核心亮点：打通本地语音输入输出。它把语音克隆与合成、Whisper 转写、全局听写、时间线编辑和代理播报组合在同一桌面应用中，并通过本地模型、REST API 与 MCP 串起输入到输出。

适合谁：需要制作多角色音频、用语音输入文字，或为 AI 代理接入本地语音输入输出的创作者与开发者
使用前准备：直接使用需下载对应平台版本；源码开发需要 Bun、Rust、Python 3.11+、Tauri 前置环境，macOS 还需 Xcode。

使用提醒：Linux 暂无预编译版本；自动粘贴目前只在 macOS 提供，情绪标签也仅由 Chatterbox Turbo 解析。

许可证：MIT

03｜jcode
在终端中管理多会话编码代理，让它们共享记忆、并行协作并处理代码与浏览器任务

它能做什么：

- 运行交互会话或单次非交互任务
- 恢复其他编码工具留下的历史会话
- 检索并整理跨会话的代理记忆
- 派生多个代理并协调消息与完成状态
- 控制 Firefox 完成网页点击、输入与截图

核心亮点：多会话代理共享记忆协作。它以持久服务管理多个终端会话，把对话提取进记忆图并按语义召回；多个代理在同一仓库工作时还能互发消息、感知文件变动，由主代理协调并行任务。

适合谁：需要长期维护编码上下文、并行调度多个代理，或在终端里连接不同模型提供方的开发者
使用前准备：需先安装 jcode，并登录或配置可用模型提供方；从源码构建需要 Git 和 Rust。

使用提醒：MCP 目前只支持 stdio 服务，HTTP 与 SSE 配置会被跳过；自我修改模式建议使用能力较强的模型，较弱模型可能造成破坏性改动。

许可证：MIT
