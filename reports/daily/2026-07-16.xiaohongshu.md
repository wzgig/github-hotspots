01｜skills
把软件工程方法拆成可组合技能，让编程 Agent 按明确流程完成开发任务

它能做什么：

- 追问需求并整理变更目标
- 建立项目术语表与决策记录
- 按红绿重构循环推进测试开发
- 诊断难复现缺陷与性能回退
- 拆分规格、工单并驱动实现审查

核心亮点：小技能组合成完整研发流程。每项技能聚焦追问、规格、测试、调试或审查等单一环节，既能由用户主动调用，也能由 Agent 按任务自动选择，方便按项目调整和组合。

适合谁：希望保留研发控制权，并让 Claude Code、Codex 等编程 Agent 遵循工程流程的开发者
使用前准备：通过 skills.sh 安装后选择目标编程 Agent，并在每个仓库运行 `/setup-matt-pocock-skills` 完成配置。

使用提醒：skills.sh 会复制可编辑文件；Claude Code 插件提供只读、随版本更新的整套技能，两种安装方式的维护方式不同。

许可证：MIT

GitHub 搜索：mattpocock/skills

02｜OpenCut
面向浏览器、桌面和移动端的开源视频编辑器，当前仓库正在重写核心架构

它能做什么：

- 开发浏览器端视频编辑界面
- 运行编辑器 API 开发服务
- 启动桌面端开发环境
- 规划接入第三方编辑器插件
- 规划通过无界面模式批量渲染

核心亮点：从同一核心构建多端编辑器。重写版本计划以 Rust 核心连接浏览器、桌面和移动端，并加入编辑器 API、插件架构、脚本页、MCP 服务与无界面自动化能力。

适合谁：想研究或参与多端视频编辑器架构的开发者；需要立即剪辑的用户应选择 README 指向的旧版本
使用前准备：先安装 proto，在仓库根目录执行 `proto use` 安装 `.prototools` 固定的开发工具。

使用提醒：当前版本仍在从头重写，README 建议现阶段使用 OpenCut Classic；架构设计期间暂不接收外部贡献。

许可证：MIT

03｜superpowers
让编程 Agent 从需求澄清到测试验收都按一套可组合的软件开发方法推进

它能做什么：

- 澄清想法并生成设计文档
- 拆分带路径和验证步骤的计划
- 隔离工作区并检查测试基线
- 按测试先行方式逐项实现
- 审查结果并提供合并或保留选项

核心亮点：流程会在任务节点自动触发。技能从需求讨论开始介入，依次推动设计确认、任务拆分、测试先行、子 Agent 实现和代码审查；相关技能会按任务自动触发，减少临时跳步。

适合谁：希望 Claude Code、Codex、Cursor 等编程 Agent 严格执行设计、测试、审查和收尾流程的开发者
使用前准备：需针对所用编程 Agent 分别安装；同时使用多个 Agent 时，每个运行环境都要单独安装。

使用提醒：通常不接收新增技能贡献，技能更新还须兼容其支持的全部编程 Agent；可选视觉组件默认会加载带版本信息的远程标识。

许可证：MIT
