01｜graphify
把代码、文档和媒体整理成可查询的知识图谱，帮 AI 助手追踪项目概念与关联

它能做什么：

- 解析代码并连接跨文件依赖关系
- 合并文档、图片和视频中的概念
- 查询与问题相关的局部子图
- 追踪任意两个概念之间的路径
- 生成交互图、摘要报告和图数据

核心亮点：用关系图替代逐文件搜索。它先用语法树解析代码，再把文档和媒体的语义概念并入同一张图；每条关系标明来自源码还是推断，便于沿路径核对项目结构。

适合谁：需要理解陌生代码库、梳理系统架构，或让 AI 助手减少逐文件搜索的开发者与团队
使用前准备：需要 Python 3.10+，并使用 uv 或 pipx 安装；文档和媒体分析还需助手模型或配置后端。

使用提醒：代码可在本地解析；文档、PDF、图片等语义分析会调用助手模型或已配置后端。PowerShell 中需使用 graphify .。

许可证：MIT

02｜awesome-llm-apps
汇集可直接克隆改造的 AI 智能体、RAG 与多智能体应用，省去从空项目搭框架

它能做什么：

- 克隆并运行不同场景的智能体模板
- 给编程助手安装可调用的技能
- 搭建多智能体协作与任务路由
- 实现文档、网页和多模态 RAG
- 参考语音、生成式界面与 MCP 案例

核心亮点：从案例目录直接挑应用骨架。项目把不同难度和交互方式的 AI 应用分目录整理，既有单文件入门智能体，也有带工具、记忆、多步推理和协作团队的完整案例。

适合谁：想快速验证 AI 应用点子、学习智能体工作流，或寻找可改造项目骨架的开发者
使用前准备：需按所选模板安装依赖，并准备模板对应的模型、API Key 或本地模型环境。

使用提醒：各模板使用的模型、数据源、外部服务和启动方式不同，运行前需进入对应目录核对说明与依赖。

许可证：Apache-2.0

GitHub 搜索：Shubhamsaboo/awesome-llm-apps

03｜Vibe-Trading
把自然语言交易问题接到市场数据、策略生成和回测流程，交付可检查的研究报告与代码

它能做什么：

- 调用市场数据完成自然语言研究
- 生成策略代码并运行跨市场回测
- 分析券商记录中的交易行为偏差
- 组织投研、量化和风控智能体协作
- 导出报告与多平台策略文件

核心亮点：研究问题一路落到回测产物。它把问题路由到数据源、金融技能和回测引擎，再补充指标、基准比较、验证结果与运行记录，让策略结论能回看对应代码和产物。

适合谁：需要研究市场、验证策略想法、复盘个人交易，或组织多角色投研流程的量化与交易研究者
使用前准备：需要 LLM API Key 或本地 Ollama；本地安装要求 Python 3.11+，也可使用 Docker。

使用提醒：实盘功能需自行授权券商并设定交易边界，可随时停止；项目不持有资金，部分连接器仅支持模拟盘和只读。

许可证：MIT
