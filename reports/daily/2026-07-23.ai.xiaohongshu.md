01｜worldmonitor
把全球新闻、地缘风险、市场与基础设施信号汇进同一张态势看板。

它能做什么：

- 汇总多类别新闻源并生成简报
- 切换三维地球与平面地图查看事件
- 关联军事、经济、灾害等信号
- 追踪股票市场、商品与加密资产
- 通过 MCP、REST API、CLI 或 SDK 调用数据

核心亮点：多源信号汇成一张态势图。项目把新闻、地图图层、风险评分和市场数据放进统一界面，并用跨流关联寻找同时出现的军事、经济、灾害与升级信号。

适合谁：需要持续观察全球事件、市场与关键基础设施的研究者、分析人员和自建情报看板团队
使用前准备：本地开发需安装 npm 依赖；应用本身可无环境变量运行，特定数据源可能需要凭据。

使用提醒：部分数据源需要凭据；MCP 的 tools/call 需要 X-WorldMonitor-Key 或 OAuth 认证。

许可证：AGPL-3.0-only（share source under AGPL-3.0-only when required）

02｜OmniRoute
把多个模型供应商收进一个本地网关，按配额、成本与健康状态自动路由和切换。

它能做什么：

- 统一暴露 OpenAI 兼容调用入口
- 按配额、成本、延迟等条件选模型
- 串联模型并在故障时自动切换
- 为编码 CLI 写入接入配置
- 通过 MCP 与 A2A 交给智能体管理

核心亮点：配额感知的模型自动接力。Auto-Combo 会实时给已连接模型评分，并在配额耗尽、服务失败或成本变化时切到下一目标；也可自定义路由策略和组合步骤。

适合谁：同时使用多家模型服务、需要统一接入编码 CLI，并希望减少配额中断的个人或团队
使用前准备：运行需 Node.js 22.x 或 24.x LTS；使用前先连接模型供应商并配置客户端接口。

使用提醒：记忆功能默认关闭；远程模式的进程启动路由仅限本机回环地址；供应商免费额度会变化。

许可证：MIT

为什么本期上榜：过去 24 小时净增 +1,657 Star（历史快照核验）
GitHub 搜索：diegosouzapw/OmniRoute

03｜openship
把代码仓库变成可运行容器，并在自有服务器上统一管理部署、域名、数据库和备份。

它能做什么：

- 检测代码栈并构建 Docker 容器
- 配置推送发布、预览环境和回滚
- 管理数据库、域名、SSL 与 CDN
- 安排数据库和数据卷备份恢复
- 通过 REST API 与 MCP 接入自动化

核心亮点：零配置文件的自托管发布。指向代码仓库后，它会识别技术栈、构建容器并配置部署；控制面可从桌面端、网页或 CLI 驱动服务器，把交付与基础设施放在同一处管理。

适合谁：想在 VPS、独立服务器或家庭实验室自托管应用，并统一管理交付流程的个人开发者和团队
使用前准备：个人模式需安装桌面应用并通过 SSH 管理服务器；团队常驻模式需在服务器安装 CLI 并完成管理员与域名向导。

使用提醒：文档仍在完善；源码 Docker Compose 方式更重，且控制面容器可访问宿主机 Docker 守护进程，只建议有明确容器化需求时使用。

许可证：Apache-2.0
