01｜code-review-graph
为 AI 编程工具建立本地代码关系图，只提供审查当前变更所需的文件与依赖上下文

它能做什么：

- 构建可增量更新的代码关系图
- 追踪变更影响的调用方、依赖和测试
- 生成带风险与测试缺口的合并请求审查
- 搜索代码实体并遍历调用与继承关系
- 导出交互图、架构概览或 Markdown Wiki

核心亮点：按变更影响范围提供上下文。它用 Tree-sitter 把仓库解析成代码关系图；文件变化后沿调用、依赖和测试边追踪影响范围，再让 AI 只读取相关文件，而非反复扫描整个项目。

适合谁：需要让 AI 审查大型仓库、分析跨文件影响或梳理陌生代码架构的开发者与研发团队
使用前准备：需要 Python 3.10+；安装后需重启编辑器或工具，再构建代码图。

使用提醒：小仓库或单文件轻改可能得不偿失；搜索排序、流程检测仍有不足，影响分析会保守地产生误报。

许可证：MIT

02｜awesome-llm-apps
把 AI Agent、RAG 和多智能体案例整理成可克隆运行的模板库，方便选型与改造

它能做什么：

- 安装可供编程代理调用的技能
- 运行单文件入门代理示例
- 复用工具、记忆与多步推理工作流
- 搭建多智能体协作任务模板
- 选择本地、云端或多模态 RAG 案例

核心亮点：按任务类型查找可运行模板。仓库把代理技能、入门代理、进阶应用、多智能体团队、语音代理和 RAG 等案例分区整理；读者可先按任务选模板，再进入子目录安装依赖并替换模型或数据源。

适合谁：想快速试跑 AI Agent、比较框架工作流，或为课程演示和原型寻找可改造样例的开发者
使用前准备：克隆示例后需安装对应 requirements.txt；部分单文件代理还需要模型 API Key。

使用提醒：不同模板使用不同模型、框架和数据源，运行前仍需进入对应目录核对依赖与配置。

许可证：Apache-2.0

GitHub 搜索：Shubhamsaboo/awesome-llm-apps

03｜wigolo
给 AI 代理提供本地网页搜索、抓取、爬取与研究入口，并保留可核对的来源证据

它能做什么：

- 并行搜索多个查询并融合排序结果
- 抓取动态网页、PDF 或指定页面区域
- 按站点地图或遍历策略爬取多页内容
- 从页面抽取表格、元数据与自定义结构
- 分解研究问题并生成带引用的资料简报

核心亮点：从网页操作到证据交付。它把搜索、抓取、爬取、抽取、缓存和研究放进同一本地工具层；结果附原文摘录、来源位置和评分，失败引擎、过期缓存与受阻页面也会明确标记。

适合谁：需要让编程代理或自托管 Agent 查网页、收集资料、抽取结构化数据并保留引用依据的开发者
使用前准备：需要 Node ≥ 20，并预留约 1.5 GB 磁盘空间。

使用提醒：处于 public beta；无 LLM 时 research、agent 与回答格式只返回较薄的原始简报，受挑战保护的网站也可能抓取失败。

许可证：GNU AGPL-3.0-only（you must publish your modified source under the same license.）
