01｜worldmonitor
把新闻、地缘事件、市场与基础设施信号汇进同一张全球态势面板

它能做什么：

- 汇总新闻源并合成为简报
- 叠加地缘、航空和基础设施地图
- 计算重点国家的局势压力指标
- 追踪交易所、商品与加密市场
- 通过 MCP、REST、CLI 或 SDK 调用数据

核心亮点：跨数据流的全球态势关联。它把新闻、军事、经济、灾害与市场数据放进统一界面，再用跨流关联和地图图层呈现信号交汇，帮助读者从分散来源建立态势概览。

适合谁：需要汇总全球新闻、市场和基础设施动态，或为 Agent 接入态势数据的分析者与开发者
使用前准备：本地开发需先安装依赖并运行开发命令；基础界面无需环境变量，部分数据源需要凭据。

使用提醒：当前桌面端 CI 发布目标仅包含 full 和 tech；部分数据源需凭据，MCP 工具调用需密钥或 OAuth。

许可证：AGPL-3.0-only（share source under AGPL-3.0-only when required）

02｜OmniRoute
把多个模型供应商收进一个兼容 OpenAI 的入口，并按额度、成本与健康状态自动切换

它能做什么：

- 接入不同模型供应商到统一端点
- 按额度、成本或延迟选择后端
- 串联模型并在故障时自动回退
- 压缩上下文并保护代码与结构化数据
- 通过 CLI、MCP 与 A2A 管理网关

核心亮点：配额感知的自动模型组合。它把已连接的模型组成自动路由池，持续参考健康状态、剩余额度、成本和延迟选择后端；当目标失败或额度耗尽时，继续转向组合中的下一项。

适合谁：需要让编程工具共用模型入口、组合免费额度，或集中维护供应商与路由规则的 AI 开发者
使用前准备：全局安装需要 Node.js 22.x 或 24.x LTS；从源码运行需复制环境文件并安装依赖。

使用提醒：新安装仅内置无密钥免费提供商；接入更多供应商仍需添加相应密钥。内存功能默认关闭。

许可证：MIT

03｜superpowers
让编程 Agent 先澄清需求、确认设计，再按计划、测试和评审推进开发

它能做什么：

- 澄清目标并分段展示设计
- 生成带文件路径和验证步骤的计划
- 以红绿重构流程先写测试
- 为每个任务派发独立子 Agent
- 完成后验证测试并给出分支选项

核心亮点：先设计确认，再让 Agent 动手。它用可组合技能约束开发顺序：先从对话提炼规格并等待确认，再拆解计划、派发子 Agent、执行测试驱动开发和代码评审，减少直接写代码导致的偏航。

适合谁：希望约束编程 Agent 工作方式，并让需求梳理、实现、测试和代码评审形成固定流程的开发者
使用前准备：需要为每个使用的编程 Agent 环境分别安装；不同环境采用各自的插件或包管理方式。

使用提醒：设计需要用户确认后才进入计划，执行也要用户说“go”；新增技能通常不接受外部贡献，修改需兼容所有受支持 Agent。

许可证：MIT
