01｜worldmonitor
把新闻、地缘风险、金融与基础设施信号汇入同一看板，帮助分析者持续掌握全球态势。

它能做什么：

- 汇总新闻源并生成 AI 简报
- 切换三维地球与平面地图
- 叠加军事、经济和灾害信号
- 追踪市场、能源、航空与基础设施
- 通过 MCP、API、CLI 或 SDK 查询数据

核心亮点：跨数据流关联全球信号。它把新闻、地图图层、市场与基础设施数据放进统一界面，再关联军事、经济、灾害和局势升级信号，便于从分散事件中观察同一地区的变化。

适合谁：适合需要追踪国际新闻、区域风险、市场变化或基础设施事件的研究者、分析人员与情报团队。
使用前准备：本地启动需克隆仓库并运行 npm install、npm run dev；特定数据源可能需要凭据。

使用提醒：基础应用无需环境变量，但特定数据源可能需要凭据；MCP 工具调用需密钥或 OAuth。

许可证：AGPL-3.0-only（Separate commercial or trademark permission needed）

02｜i-have-adhd
让编码助手先给行动结论，再用编号步骤说明操作，减少寒暄、旁支和埋在长文里的答案。

它能做什么：

- 先输出当前最该执行的动作
- 把多步骤任务整理成编号清单
- 压缩与当前任务无关的旁支
- 用明确时间代替模糊耗时描述
- 在结尾留下一个具体下一步

核心亮点：把答案从长文里提到最前。这项技能用一组固定输出规则约束编码助手：行动优先、步骤编号、减少旁支和客套，并在每轮结束时给出一个可继续执行的动作。

适合谁：适合容易被冗长回答打断思路，或希望 Claude Code、Codex 直接给出可执行步骤的使用者。
使用前准备：需先在 Claude Code 或 Codex 安装插件，再通过对应命令启用输出风格。

使用提醒：只调整编码助手的输出风格；规则会压制旁支、取消寒暄，并把列表限制在 5 项。

许可证：MIT

03｜OmniRoute
把多个模型供应商接到同一入口，并在额度耗尽或调用失败时自动切换后备模型。

它能做什么：

- 汇集不同供应商的模型接口
- 按策略选择当前请求的模型
- 在限额或故障时切换后备模型
- 压缩提示词与工具输出以减少用量
- 通过 MCP 和 A2A 交给智能体管理

核心亮点：用模型组合自动接力请求。核心工作流是把多个模型排成组合，并依据额度、成本、延迟、成功率等状态实时选择；当前目标不可用时，请求会转向组合中的下一个模型。

适合谁：适合同时使用多个模型服务，并希望为编码工具统一接口、分配额度和准备后备线路的 AI 开发者与团队。
使用前准备：全局安装可用 npm；源码运行要求 Node.js 22.x 或 24.x LTS，并连接至少一个模型供应商。

使用提醒：记忆功能默认关闭；远程令牌有作用域，启动进程的路由仅允许本机回环访问。

许可证：MIT
