01｜Vibe-Trading
把交易问题变成可执行的研究、回测与报告，并可接入经纪商完成受限交易。

它能做什么：

- 用自然语言发起市场研究与策略分析
- 生成策略代码并运行跨市场回测
- 解析券商交易记录并诊断行为偏差
- 组织多智能体团队完成投资与风险评审
- 导出报告及 TradingView、TDX、MT5 产物

核心亮点：交易研究全流程工作台。同一工作区把自然语言问题路由到市场数据、策略生成、回测验证和报告导出，并用持久记忆保存可复用的研究流程。

适合谁：想把交易想法落成研究与回测的量化研究者，也适合需要复盘个人交易记录或编排多角色评审的用户。
使用前准备：需要受支持的 LLM API Key，或本地运行 Ollama；本地安装要求 Python 3.11+，Docker 路径需 Docker。

使用提醒：自主交易仅在用户自行授权经纪商并设定限制后启用；项目不托管资金，可随时停止。历史数据加载器不覆盖实时盘口。

许可证：MIT

02｜graphify
把代码、文档和媒体整理成可查询的知识图谱，替代逐文件搜索和反复通读。

它能做什么：

- 扫描项目并生成统一知识图谱
- 查询概念并返回相关子图
- 追踪两个节点之间的最短路径
- 解释单个节点的来源与连接
- 合并代码、文档、PDF、图像和视频信息

核心亮点：从文件搜索切到图谱追踪。代码通过 tree-sitter 在本地解析，跨文件关系汇入 graph.json；查询、路径和节点解释直接沿图遍历，并标明源码关系与推断关系。

适合谁：维护大型代码库、做架构梳理或接手陌生项目的开发者，也适合需要把代码、数据库结构和资料放进同一图谱的团队。
使用前准备：需要 Python 3.10+；推荐使用 uv，也可用 pipx。处理 PDF、Office、视频等内容需安装对应可选组件。

使用提醒：代码解析可完全本地完成；文档、PDF 和图像的语义提取会调用所选模型，除非改用本地后端。部分格式需额外组件。

许可证：MIT

03｜awesome-llm-apps
提供可直接运行和改造的 LLM 应用模板，省去重复搭建 Agent、RAG 与 MCP 基础流程。

它能做什么：

- 克隆模板快速启动一个 AI Agent
- 改造现有 Agent 循环和工具调用
- 搭建多智能体协作任务
- 组合检索、记忆与多源 RAG 流程
- 参考语音、生成式界面和微调示例

核心亮点：按场景挑选可运行模板。仓库按 Agent、常驻任务、多智能体、语音、MCP、RAG、记忆和微调分类；每个示例独立放置源码，可从相近场景直接克隆和定制。

适合谁：准备学习或验证 LLM 应用思路的开发者，尤其适合从可运行样例起步，再替换模型、数据源和业务逻辑的场景。
使用前准备：需先克隆仓库，进入目标模板目录并安装 requirements.txt；部分示例需要相应模型服务的 API Key。

使用提醒：各模板是彼此独立的起步代码，需按子目录配置依赖、模型和密钥，再结合自己的业务继续定制。

许可证：Apache-2.0
