01｜worldmonitor
把全球新闻、地缘风险、市场与基础设施信号汇集到同一套态势看板

它能做什么：

- 汇总多类别新闻源并生成简报
- 切换立体地球与平面地图查看信号
- 关联军事、经济、灾害与升级迹象
- 追踪交易所、商品和加密资产动态
- 通过 MCP、REST、CLI 或 SDK 取用数据

核心亮点：把分散信号汇成态势视图。项目把新闻、地图图层、风险评分和市场雷达放入统一界面，并关联不同数据流，帮助读者从单条事件继续查看区域压力与相关变化。

适合谁：需要持续追踪国际新闻、地缘风险、市场变化或基础设施状态的研究者与分析人员
使用前准备：本地启动需执行 npm install 和 npm run dev；特定数据源可能需要凭据。

使用提醒：MCP 的 tools/call 需要 X-WorldMonitor-Key 或 OAuth；部分数据源还需凭据。

许可证：AGPL-3.0-only（Separate commercial or trademark permission needed）

02｜OmniRoute
用一个兼容接口连接多家 AI 服务，并按配额、成本或状态自动切换模型

它能做什么：

- 接入兼容 OpenAI 接口的编码工具
- 按配额、成本、延迟或状态选择模型
- 串联多个模型组成自动回退链
- 压缩提示词与工具输出以减少令牌
- 通过 MCP 或 A2A 让智能体管理网关

核心亮点：会自动换路的模型组合。Combo 把多个模型排成可配置链路；当配额不足、服务失败或成本变化时，路由器按所选策略切到下一目标，也可让多模型并行后汇总答案。

适合谁：同时使用多家模型供应商，并希望统一配置编码工具、控制配额或降低上下文消耗的 AI 开发者
使用前准备：运行时要求 Node.js 22.x 或 24.x LTS；安装后执行 omniroute 启动服务。

使用提醒：内存功能默认关闭；原生 SQLite 不可用时会回退到纯 JavaScript 引擎；目前没有独立的仅 CLI 包。

许可证：MIT

03｜RuView
把 WiFi 信号变化转换为无摄像头的室内存在、动作与生命体征数据

它能做什么：

- 检测人员存在、进出与房间占用
- 估算静坐或睡眠时的呼吸和心率
- 识别行走、坐姿、手势与跌倒
- 生成房间射频指纹并发现环境变化
- 接入 Home Assistant 与 Matter 生态

核心亮点：用无线反射感知室内活动。ESP32 节点采集信道状态信息，也就是人体影响无线电波后的细微变化；信号处理与模型再将其转成存在、动作、姿态和生命体征结果。

适合谁：研究无摄像头室内感知，或为智能家居、照护与空间管理搭建 WiFi 传感系统的开发者
使用前准备：高级感知需要能提供 CSI 的 ESP32-S3 或研究网卡；Docker 仅用模拟数据评估。

使用提醒：仍属 Beta 软件；API 和固件可能变化，单个 ESP32 空间分辨率有限，部分姿态能力仍待数据收集与评估。

许可证：MIT
