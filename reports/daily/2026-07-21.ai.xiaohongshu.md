01｜code-review-graph
为 AI 编程工具建立本地代码结构图，只把审查相关的文件和依赖送入上下文

它能做什么：

- 解析函数、类、导入和调用关系
- 定位代码改动的影响范围
- 生成面向审查的精简上下文
- 搜索跨文件与跨仓库的代码实体
- 导出可视化图、Wiki 和多种图格式

核心亮点：按影响范围裁剪审查上下文。它用 Tree-sitter 解析仓库并保存函数、调用、继承和测试关系；文件变化后沿图追踪受影响范围，让 AI 优先读取相关代码，而不是反复扫描整个项目。

适合谁：适合在大型仓库或多文件改动中做代码审查、影响排查和架构梳理的开发者与团队
使用前准备：需要 Python 3.10+；安装后需重启编辑器或 AI 工具

使用提醒：小型仓库、简单单文件改动和一次性问题未必适合；影响分析会保守多报，搜索排序与流程检测仍有不足

许可证：MIT

02｜voicebox
把声音克隆、语音生成、全局听写和 AI 代理播报放进一套本地桌面工具

它能做什么：

- 克隆参考音频中的声音
- 把长文本分段合成为连续语音
- 用全局快捷键录音并转写文字
- 为生成语音叠加音高、混响等效果
- 让 MCP 代理调用指定声音播报

核心亮点：打通本地语音输入与输出。它把 TTS、Whisper 转写、本地语言模型和声音档案连成一套流程：用户可以听写内容、润色文本、生成语音，也能让接入 MCP 的代理调用同一套声音。

适合谁：适合制作配音、播客和互动角色，或希望用语音输入并让 AI 代理以指定声音播报的人
使用前准备：从源码开发需 Bun、Rust、Python 3.11+ 与 Tauri 前置组件；macOS 还需 Xcode

使用提醒：Linux 暂无预编译版本；全局听写自动粘贴目前明确覆盖 macOS；拟声标签仅由 Chatterbox Turbo 解析

许可证：MIT

03｜openship
把代码仓库自动构建成容器并部署到云端或自有服务器，同时管理配套基础设施

它能做什么：

- 检测仓库技术栈并构建容器
- 按代码推送触发部署和预览环境
- 配置数据库、存储与后台任务
- 管理域名、证书、邮件和备份
- 查看构建日志、容器指标和资源用量

核心亮点：同一界面部署到不同服务器。它把代码构建、容器部署和基础设施管理放在同一平台，并提供桌面端、网页端、命令行、REST API 与 MCP；工作负载可运行在托管云或自有 Linux 服务器。

适合谁：适合需要自托管应用、维护预览与正式环境，或统一管理服务和基础设施的个人开发者与团队
使用前准备：自托管需可用的 Linux 服务器；使用 Docker Compose 时需复制 .env.example 为 .env

使用提醒：文档仍在补全；多节点集群、负载均衡界面、私有网络和高级监控等能力尚在后续计划中

许可证：Apache-2.0
