01｜Vibe-Trading
把交易问题变成市场研究、策略代码、回测结果和可复查报告

它能做什么：

- 提出问题并调用数据与文档完成研究
- 生成策略代码并运行跨市场回测
- 解析券商记录并诊断交易行为
- 提取交易规则并生成影子账户报告
- 导出报告及多平台策略文件

核心亮点：从问题到可复查研究产物。它把自然语言请求接到数据加载、策略生成、回测、验证和报告流程，保留运行卡片、工具轨迹与导出文件，方便继续检查和迭代。

适合谁：需要验证交易想法、复盘个人交易或搭建量化研究流程的投资研究者与开发者
使用前准备：需要受支持模型的 API Key，或使用本地 Ollama；本地安装要求 Python 3.11+，也可使用 Docker。

使用提醒：主要面向研究、模拟与回测；实盘交易需自行授权券商并设置交易边界，相关能力标注为实验性，使用风险由用户承担。

许可证：MIT

02｜awesome-llm-apps
从现成模板起步，搭建可运行的 AI Agent、RAG 与多智能体应用

它能做什么：

- 克隆模板并替换配置运行示例
- 为编程助手安装可复用技能
- 搭建单智能体和多智能体工作流
- 实现文档、网页与多模态 RAG
- 组合语音、MCP 与生成式界面

核心亮点：按真实场景组织的应用模板库。仓库不是单一框架，而是按技能、入门 Agent、复杂团队、语音、MCP、RAG 和生成式界面分类的示例集合，可选择一个场景后直接运行和改造。

适合谁：想快速学习、改造或交付 AI Agent 与 RAG 应用的开发者，适合从具体场景模板开始实践
使用前准备：按所选模板安装 requirements.txt；入门 Agent 通常需要对应模型的 API Key。

使用提醒：各模板依赖的模型、服务和运行方式不同，需要进入具体目录核对配置；部分条目指向外部仓库。

许可证：Apache-2.0

03｜superpowers
让编程 Agent 先澄清需求，再按设计、计划、测试和审查流程交付代码

它能做什么：

- 追问需求并分段提交设计确认
- 拆解带路径和验证步骤的实施计划
- 按测试驱动流程完成最小实现
- 分派子 Agent 执行并审查任务
- 验证测试后给出分支收尾选项

核心亮点：把开发纪律装进 Agent。它用一组自动触发、可组合的技能串起头脑风暴、计划、测试驱动开发、子 Agent 执行、代码审查和分支收尾，让 Agent 按已确认流程推进。

适合谁：希望减少编程 Agent 盲目写代码，并让需求确认、实现和验收过程更可控的开发者与团队
使用前准备：需要在所用编程 Agent 中单独安装；同时使用多个客户端时，每个客户端都要分别安装。

使用提醒：通常不接受新增技能贡献，技能更新需兼容支持的编程 Agent；可选视觉组件默认会发送所用版本信息，可通过环境变量关闭。

许可证：MIT
