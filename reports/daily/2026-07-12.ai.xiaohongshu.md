# 正文

2026-07-12 · D001 · AI 专题榜

AI 项目的名字往往很抽象。这一期不堆术语，只讲清它替谁做什么、能完成哪些任务，以及是否值得加入你的工具箱。

先看结论：
- superpowers：为编程 Agent 配上从需求澄清、设计确认到测试和代码审查的开发流程
- firecrawl：把网页搜索、单页抓取和整站遍历接入接口，返回 Markdown、截图或结构化数据
- hermes-agent：让 AI Agent 在终端和消息平台持续工作，并把任务经验沉淀成记忆与技能

01｜superpowers
为编程 Agent 配上从需求澄清、设计确认到测试和代码审查的开发流程

它能做什么：

- 澄清粗略想法并保存设计文档
- 创建隔离工作区并验证测试基线
- 拆分带文件路径和验证步骤的任务
- 调度子 Agent 分工实现并分阶段审查
- 完成测试后提供分支处理选项

核心亮点：从需求确认到交付的技能链。技能按场景自动触发：先澄清需求、确认设计和编写计划，再分派实现任务，通过测试、代码审查和完成检查逐步推进。

适合谁：希望编程 Agent 遵循明确研发流程，而不是直接生成代码的开发者和团队
使用前准备：需要在所用的编程 Agent 环境中安装；使用多个环境时须分别安装。

使用提醒：设计和执行启动前需要用户确认；项目通常不接受新增技能，技能更新须兼容其支持的全部编程 Agent。

许可证：MIT

02｜firecrawl
把网页搜索、单页抓取和整站遍历接入接口，返回 Markdown、截图或结构化数据

它能做什么：

- 搜索网页并获取结果页面正文
- 转换网址为 Markdown、HTML 或 JSON
- 发现网站内的全部页面链接
- 批量抓取多个网址或遍历整站
- 按提示操作动态页面后提取数据

核心亮点：从网页发现到交互式提取。同一套接口覆盖搜索、单页抓取、整站遍历、网址发现和批量任务；处理动态页面时，还能先点击、滚动或输入，再返回整理后的内容。

适合谁：需要为 AI Agent、搜索应用或数据管道持续接入网页内容的开发者
使用前准备：使用托管服务需注册并取得 API Key；也可按自托管指南在本地部署。

使用提醒：最终用户须遵守目标网站政策、隐私政策和使用条款；默认遵循 robots.txt 指令。

许可证：AGPL-3.0（The SDKs and some UI components are licensed under the MIT License.）

03｜hermes-agent
让 AI Agent 在终端和消息平台持续工作，并把任务经验沉淀成记忆与技能

它能做什么：

- 切换不同模型提供商而无需修改代码
- 搜索历史会话并整理跨会话记忆
- 创建可在使用中持续改进的技能
- 调度定时任务并向消息平台发送结果
- 并行派出隔离的子 Agent 处理工作流

核心亮点：经验转技能的闭环学习。Agent 会保存和检索会话记忆，在复杂任务后创建技能，并在后续使用中持续改进；同一工作还能在终端和多个消息平台间延续。

适合谁：需要让个人 AI 助手长期记忆、远程工作或定时处理任务的用户与开发者
使用前准备：安装器会准备 uv、Python 3.11、Node.js、ripgrep、ffmpeg 和 Git Bash；运行还需配置模型提供商。

使用提醒：命令审批、私信配对和容器隔离需按安全设置配置；完整语音依赖目前与 Android 不兼容。

许可证：MIT
