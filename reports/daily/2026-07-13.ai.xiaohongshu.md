01｜awesome-llm-apps
把重复搭建的 Agent、RAG 与 MCP 流程变成可直接运行和改造的 LLM 应用模板。

它能做什么：

- 克隆独立模板并改造成 LLM 应用
- 切换 Claude、Gemini、GPT 等模型
- 搭建单智能体、多智能体与常驻任务
- 组合 RAG、语音、MCP 和记忆工作流
- 跟随配套教程运行和修改模板

核心亮点：从可运行模板开始搭应用。每个模板自带完整源码和独立运行结构，覆盖智能体循环、RAG、MCP、语音及多智能体协作，可先跑通示例，再替换模型与业务逻辑。

适合谁：想快速验证 LLM 应用构想，或需要现成代码搭建智能体、RAG 和多智能体流程的开发者。
使用前准备：按所选模板配置对应模型或服务的 API Key，并安装 requirements.txt 中声明的依赖。

许可证：Apache-2.0

为什么本期上榜：过去 24 小时净增 +710 Star（历史快照核验）
GitHub 搜索：Shubhamsaboo/awesome-llm-apps

02｜hermes-agent
让 AI 助手跨终端和聊天平台持续工作，并从任务经验中生成、改进可复用技能。

它能做什么：

- 从复杂任务中自动创建并改进技能
- 搜索历史会话并延续跨会话记忆
- 通过终端或消息平台继续同一对话
- 按自然语言计划定时执行并投递任务
- 启动隔离子智能体并行处理工作流

核心亮点：会积累经验的技能循环。它把持久记忆、会话搜索、用户建模和技能系统连成闭环：任务完成后沉淀技能，后续使用中继续改进，并在新会话里调用过去经验。

适合谁：需要让个人 AI 助手长期运行、跨设备接续对话，或定时处理多步骤任务的开发者和个人用户。
使用前准备：需选择并配置一个模型提供商；使用消息网关时还需配置对应聊天平台。

使用提醒：Termux 需使用专用安装额外项，完整依赖目前包含 Android 不兼容的语音组件。

许可证：MIT

为什么本期上榜：过去 24 小时净增 +478 Star（历史快照核验）
GitHub 搜索：NousResearch/hermes-agent

03｜Vibe-Trading
把自然语言交易问题串成数据获取、策略生成、回测验证和报告交付的研究流程。

它能做什么：

- 把自然语言问题转成可运行分析
- 生成策略代码并执行跨市场回测
- 读取券商流水并诊断交易行为偏差
- 组织多智能体完成投研与风险复核
- 导出报告及 Pine Script、TDX、MQL5 产物

核心亮点：从问题到可检查研究产物。请求先路由到数据、技能和工具，再生成策略、运行匹配的回测引擎，并补充指标、基准、验证产物和报告，方便后续复核或导出。

适合谁：需要研究交易想法、复盘个人流水、比较策略或组织多角色投研流程的量化研究者和交易开发者。
使用前准备：需要受支持模型的 API Key，或本地 Ollama；本地安装要求 Python 3.11+，也可使用 Docker。

使用提醒：自主交易仅在用户授权的经纪商和设定限额内运行；项目不托管资金，历史数据加载器不处理实时逐笔与订单簿。

许可证：MIT
