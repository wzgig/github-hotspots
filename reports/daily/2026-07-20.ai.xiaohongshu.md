01｜code-review-graph
为代码库建立持续更新的结构图，让 AI 审查变更时只读取相关代码与测试。

它能做什么：

- 解析函数、类、导入与调用关系
- 计算代码改动的影响范围
- 定位受影响的执行流程与测试缺口
- 搜索跨仓库的代码实体与关联
- 导出图谱、架构文档与 Markdown Wiki

核心亮点：用代码关系图缩小审查范围。它用 Tree-sitter 解析仓库，把函数、类、导入和调用存成关系图；文件变化时只更新相关部分，再经 MCP 返回审查所需的最小代码范围。

适合谁：适合用 AI 审查合并请求、分析跨文件影响或梳理大型代码库架构的开发者与团队。
使用前准备：需 Python 3.10+；安装并构建图谱后，应重启编辑器或编码工具。

使用提醒：小型单文件改动可能比直接读文件更重；搜索排序及部分语言的流程识别仍需改进，影响分析也可能多报候选文件。

许可证：MIT

02｜ai-agent-book
把 AI Agent 的设计原理、工程方法和按章实验放在同一仓库，供读者边读边实践。

它能做什么：

- 阅读中文、英文与泰米尔语电子书
- 运行按章节组织的 Agent 示例
- 对比不同上下文与检索策略
- 搭建感知、执行和协作类 MCP 工具
- 复现实验并修改配套代码

核心亮点：按章节贯通原理与实验。全书以“Agent = LLM + 上下文 + 工具”为主线，正文解释设计方法，配套目录把概念落到可运行示例、复现指南或设计文档中。

适合谁：适合希望系统学习 Agent 原理，并通过代码实验理解上下文、工具、评估和协作机制的读者。
使用前准备：自行编译 PDF 需安装 pandoc、xelatex、ElegantBook 文档类与相关字体；运行部分示例需配置 API Key。

使用提醒：英文与泰米尔语内容可能滞后于中文原版；部分训练、评测和机器人实验依赖外部仓库或真实硬件，部分项目仅有设计文档。

许可证：Apache-2.0（部分子项目可能包含各自的许可证信息，请以子项目中的说明为准。）

03｜wigolo
给 AI Agent 一套本地网页工具，完成搜索、抓取、爬取、提取与带引用的研究整理。

它能做什么：

- 并行搜索多个查询与来源
- 抓取动态网页、PDF 和指定页面区域
- 爬取站点并提取表格或结构化数据
- 缓存页面并检索相似内容
- 拆解问题并生成带引用的研究材料

核心亮点：让 Agent 获得本地网页工作台。它把搜索、抓取、爬取、提取和缓存放进同一套 MCP、REST 与 SDK 接口；结果附原文位置、评分和失败状态，方便 Agent 判断证据质量。

适合谁：适合需要让编码助手或自托管 Agent 搜网页、整理来源、监控页面变化并保留本地缓存的开发者。
使用前准备：需 Node ≥ 20，并预留约 1.5 GB 磁盘空间；核心工具无需 API Key。

使用提醒：项目仍处于 public beta；research、agent 和回答合成功能若不配置 LLM，只返回原始简报与证据；受挑战保护的网站可能无法读取。

许可证：GNU AGPL-3.0-only（you must publish your modified source under the same license.）
