01｜system_prompts_leaks
按厂商、产品和版本归档 AI 系统提示词，方便检索指令结构并对照不同实现

它能做什么：

- 检索不同厂商与模型的系统提示词
- 对照同一产品不同版本的指令变化
- 查看编程助手、搜索和办公集成提示词
- 查阅工具定义、策略与原始提示词资料
- 定位近期更新的提示词条目

核心亮点：跨产品提示词档案。仓库按厂商和产品建立目录，并细分模型版本、工具、集成和旧版资料，便于从同一入口查找提示词并横向对照指令设计。

适合谁：研究模型行为、提示词设计或 AI 产品交互规则的读者

许可证：CC0-1.0

02｜OfficeCLI
用单个命令行程序创建、读取和修改办公文档，并为脚本或 AI Agent 提供结构化操作接口

它能做什么：

- 创建并编辑三类常见办公文档
- 读取文本、结构、样式、公式与图表
- 渲染 HTML、PNG 和实时浏览器预览
- 合并模板占位符与结构化数据
- 批量执行操作并输出结构化结果

核心亮点：可视化文档操作闭环。内置渲染器把文档转成 HTML 或 PNG，Agent 可先修改元素，再查看页面效果并继续修正布局，形成“编辑、观察、调整”的反馈流程。

适合谁：需要在脚本、容器或 AI Agent 工作流中自动处理办公文件的开发者
使用前准备：直接运行发行版无需安装 Office 或 .NET；从源码编译需要 .NET 10 SDK。

使用提醒：驻留模式会延迟写入磁盘；其他程序读取文件前需先执行 save 或 close。

许可证：Apache-2.0

03｜strix
让多个 AI 渗透测试 Agent 动态运行目标、验证漏洞，并输出复现线索、修复建议和报告

它能做什么：

- 扫描本地代码、代码仓库或 Web 应用
- 验证访问控制、注入与业务逻辑漏洞
- 协调多个 Agent 并行完成渗透测试
- 输出漏洞复现步骤与修复建议
- 接入持续集成检查代码变更

核心亮点：以实际利用验证漏洞。Agent 不只读取代码或匹配规则，还会运行目标并尝试构造概念验证；确认问题后整理复现步骤、修复建议，并可生成补丁和报告。

适合谁：获授权检查本地代码、仓库或 Web 应用的安全团队与开发团队
使用前准备：需要运行中的 Docker，以及受支持模型供应商提供的 LLM API Key。

使用提醒：只能测试自己拥有或已获许可的应用；使用者需自行承担合规与合法使用责任。

许可证：Apache-2.0

04｜meetily
在本机捕获并实时转写会议，再用可选模型生成摘要，避免把录音和转写内容发送到云端

它能做什么：

- 捕获会议并实时生成文字记录
- 使用 Whisper 或 Parakeet 本地转写
- 导入已有音频生成或增强转写
- 选择本地或外部模型生成会议摘要
- 在本地保存录音、模型和转写内容

核心亮点：会议数据留在本机。音频采集、转写和文件存储都可在本机完成；摘要既可选择 Ollama 本地模型，也可接入其他供应商或自建兼容接口。

适合谁：需要离线整理敏感会议记录，并希望掌控录音、转写和摘要数据的个人或组织
使用前准备：Windows 与 macOS 可安装发行版；Linux 需按文档从源码构建。源码开发需要 Rust 和 Node.js。

使用提醒：社区版的自动说话人区分未在 README 中列为现有能力，而是说明计划加入另一套 PRO 产品。

许可证：MIT

05｜herdr
把多个编程 Agent 放进终端分屏中统一查看、切换和恢复，并让 Agent 通过接口互相等待或创建面板

它能做什么：

- 查看 Agent 的阻塞、工作与完成状态
- 拆分并拖动多个真实终端面板
- 分离后从其他终端或 SSH 重新连接
- 让 Agent 创建面板并读取其他任务输出
- 通过插件扩展面板与终端工作流

核心亮点：终端里的 Agent 复用器。它把每个 Agent 保留为真实终端会话，并提供分屏、分离重连、状态总览和 socket API，使人和 Agent 都能组织并等待并行任务。

适合谁：同时运行多个终端编程 Agent，并需要远程恢复会话或协调任务的开发者
使用前准备：安装发行版后在工作目录运行 herdr；从源码构建需要 Rust 工具链。

使用提醒：Windows 版本在 README 中标注为 beta。

许可证：AGPL-3.0-or-later（commercial licenses are available for organizations that cannot comply with AGPL）

06｜orca
在隔离的 Git 工作树中并行运行多个命令行编程 Agent，并集中查看终端、代码差异和任务状态

它能做什么：

- 把同一任务分发给多个独立 Agent
- 比较各工作树结果并合并选定方案
- 在代码差异行上批注并反馈给 Agent
- 连接远程主机运行 Agent 与终端
- 从手机监控任务并继续发送指令

核心亮点：并行工作树对比方案。同一提示可分发到多个 Agent，每个 Agent 使用独立 Git 工作树；开发者集中查看终端和差异，比较结果后再选择并合并方案。

适合谁：需要比较多个 Agent 方案、并行推进代码任务并集中审查结果的开发者
使用前准备：需安装 macOS、Windows 或 Linux 桌面版；所用编程 Agent 需能在终端运行。

使用提醒：移动端用于配合桌面应用监控和引导 Agent，不是 README 所述的独立桌面替代品。

许可证：MIT

07｜OmniRoute
在本机提供统一模型接口，把编程工具请求路由到不同供应商，并在额度或连接失败时切换后备模型

它能做什么：

- 把不同客户端协议转换为统一接口
- 组合多个模型并配置路由策略
- 监测连接状态并跳过故障目标
- 压缩输入和工具输出以减少令牌占用
- 通过 MCP 或 A2A 暴露网关操作

核心亮点：模型组合与自动回退。用户把供应商和模型串成组合，再按优先级、成本、额度或状态选择目标；某个连接不可用时，路由器会尝试组合中的下一项。

适合谁：需要为多个 AI 工具统一管理模型连接、额度、成本和后备路线的开发者
使用前准备：全局安装需要 Node.js 22.x 或 24.x LTS；也可通过 Docker、桌面版或源码运行。

使用提醒：模型免费额度、价格和可用性取决于外部供应商；使用付费服务时费用直接由供应商收取。

许可证：MIT
