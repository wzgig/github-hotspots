01｜system_prompts_leaks
汇集多家 AI 聊天产品、编程工具及历史版本的系统提示词，方便集中查阅和横向对照

它能做什么：

- 查阅不同厂商与产品的提示词
- 对照同一模型的不同版本
- 查看编程与办公助手的指令
- 定位工具定义和个性配置
- 从更新表找到新增提示词

核心亮点：跨产品提示词资料库。README 按厂商、模型和产品形态整理提示词，并补充旧版本、原始稿、工具定义及近期更新入口，便于沿产品或版本关系进行对照。

适合谁：需要研究模型指令、工具约束或不同 AI 产品交互设计的读者

许可证：CC0-1.0

02｜OfficeCLI
通过单个命令行程序创建、读取和修改 Word、Excel、PowerPoint 文件，无需安装 Office

它能做什么：

- 创建并编辑常用 Office 文件
- 读取文本、样式、公式和图表
- 渲染文档并预览编辑结果
- 用 JSON 填充跨格式模板
- 导出结构并批量重建文档

核心亮点：可视化文档修改闭环。内置渲染引擎把文档转为 HTML 或逐页 PNG，配合实时预览，让 Agent 修改元素后检查版面，再继续处理溢出与重叠问题。

适合谁：需要在脚本、容器、流水线或 AI Agent 中自动处理办公文档的开发者
使用前准备：直接使用发布程序无需 Office 和额外运行时；从源码编译需要 .NET 10 SDK。

使用提醒：常驻模式会延迟写盘；其他程序读取文件前需先执行 save 或 close 刷新磁盘文件。

许可证：Apache-2.0

为什么本期上榜：GitHub Trending 显示过去 7 天 +5,789 Star
GitHub 搜索：iOfficeAI/OfficeCLI

03｜strix
让多个 AI 渗透测试 Agent 运行目标、验证漏洞，并交付复现证据、修复建议和报告

它能做什么：

- 扫描代码库、仓库或 Web 应用
- 验证漏洞并生成复现步骤
- 分析访问控制与注入问题
- 生成修复补丁和测试报告
- 在拉取请求中检查变更范围

核心亮点：用实际利用验证漏洞。Agent 会动态运行目标，并组合侦察、浏览器、终端和漏洞利用工具；发现问题后生成可工作的概念验证与复现步骤，减少仅靠静态规则产生的误报。

适合谁：获得明确测试授权后，需要检查本地代码、代码仓库或 Web 应用的开发与安全团队
使用前准备：需要正在运行的 Docker，以及受支持供应商提供的 LLM API key。

使用提醒：只能测试自己拥有或已获许可的应用；使用者需自行承担合规与合法使用责任。

许可证：Apache-2.0

为什么本期上榜：GitHub Trending 显示过去 7 天 +6,443 Star
GitHub 搜索：usestrix/strix

04｜herdr
在终端集中查看、拆分和恢复多个 Agent 会话，让任务离开当前终端后继续运行

它能做什么：

- 查看多个 Agent 的终端输出
- 拆分窗格并用键鼠操作
- 分离会话后重新连接
- 通过套接字创建和读取窗格
- 安装插件扩展工作流

核心亮点：会话分离后持续运行。herdr 把多个 Agent 放进可拆分的终端窗格，并保留会话状态；用户离开当前终端后，可从其他终端或 SSH 重新连接继续查看和操作。

适合谁：同时运行多个终端编程 Agent，并需要远程接回或保留会话状态的开发者

许可证：AGPL-3.0-or-later（commercial licenses are available for organizations that cannot comply with AGPL）

为什么本期上榜：GitHub Trending 显示过去 7 天 +4,714 Star
GitHub 搜索：ogulcancelik/herdr

05｜codex-plugin-cc
把本机 Codex 接入 Claude Code，用斜杠命令完成代码审查、任务委派和会话移交

它能做什么：

- 审查当前改动或分支差异
- 挑战设计假设和风险
- 委派排错与修复任务
- 查看、获取或取消后台任务
- 把当前对话移交给 Codex

核心亮点：双助手任务交接。插件通过本机 Codex CLI 和 app server 工作：Claude Code 发起审查或后台任务，再查询结果；切换工具时，还能把当前对话导入持久 Codex 线程。

适合谁：已在 Claude Code 中工作，希望调用 Codex 复核代码或接手开发任务的工程师
使用前准备：需要 ChatGPT subscription 或 OpenAI API key，并安装 Node.js 18.18 or later。

使用提醒：普通审查和挑战式审查均为只读；审查门可能形成长时间循环并快速消耗使用额度。

许可证：Apache-2.0

为什么本期上榜：GitHub Trending 显示过去 7 天 +4,484 Star
GitHub 搜索：openai/codex-plugin-cc

06｜orca
把多种命令行编程 Agent 放进统一桌面工作区，用隔离 worktree 并行尝试、比较和合并结果

它能做什么：

- 并排运行多种编程 Agent
- 把同一任务分发到隔离 worktree
- 比较结果并合并选中实现
- 在差异行添加反馈
- 通过手机监控和引导任务

核心亮点：隔离分支并行试做。Orca 为每个 Agent 分配独立 Git worktree，可把同一提示分发给多个 Agent；完成后在统一界面比较差异、添加评语，并选择结果合并。

适合谁：需要并行尝试多种实现，并集中审查 Agent 代码差异的个人开发者或研发团队
使用前准备：桌面端提供 macOS、Windows 和 Linux 构建；需另行具备要运行的命令行 Agent。

使用提醒：手机端是桌面应用的配套工具，用于监控和引导 Agent，需要与桌面端配对。

许可证：MIT

为什么本期上榜：GitHub Trending 显示过去 7 天 +4,328 Star
GitHub 搜索：stablyai/orca

07｜OmniRoute
在本机建立统一 AI 接口，把编程工具请求转发到不同模型供应商，并在限额或故障时切换目标

它能做什么：

- 把不同模型供应商接到统一接口
- 按策略组合模型并自动路由
- 在配额或连接失败时切换目标
- 压缩部分输入和工具输出
- 查看用量、成本、健康和日志

核心亮点：组合模型自动回退。用户把多个供应商或账号编成路由链，系统依据配额、成本、延迟和连接健康度选择目标；遇到限流或故障时，跳过当前连接并尝试后备项。

适合谁：需要集中维护多个模型账号、供应商连接和编程工具配置的 AI 开发者或团队
使用前准备：全局安装需 Node ≥22.0.0；从源码运行需复制环境配置并安装 npm 依赖。

使用提醒：免费额度、价格和供应商条款来自各上游；README 提醒应在面板查看条款标记和剩余额度。

许可证：MIT
