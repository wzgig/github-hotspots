# 标题

GitHub AI 周报 W002｜7 个项目到底能做什么

# 正文

2026-07-19 · W002 · AI 专题榜

AI 项目的名字往往很抽象。这一期不堆术语，只讲清它替谁做什么、能完成哪些任务，以及是否值得加入你的工具箱。

先看结论：
- awesome-llm-apps：从可运行模板入手搭建 AI Agent、RAG、语音与多智能体应用
- orca：把多个终端编程 Agent 放进隔离工作区，并行执行、比较和审查结果
- OfficeCLI：让脚本和 AI Agent 在无 Office 环境中创建、读取与修改办公文档
- graphify：把代码、文档和媒体整理成可查询的知识图谱，减少逐文件搜索
- superpowers：让编程 Agent 先澄清需求、确认设计，再按测试与审查流程完成开发
- Vibe-Trading：把自然语言交易问题转成市场研究、策略回测与可检查的报告产物
- hallmark：给编程 Agent 一套网页设计规则，生成、审查或重做更有区分度的界面

01｜awesome-llm-apps
从可运行模板入手搭建 AI Agent、RAG、语音与多智能体应用

它能做什么：

- 安装项目复盘等编程 Agent 技能
- 克隆并运行旅行、研究等入门 Agent
- 搭建文档问答与多来源 RAG 流程
- 组合多个 Agent 完成研究或业务任务
- 生成卡片、图表等交互式 Agent 界面

核心亮点：按任务挑选可运行模板。仓库把示例分成技能、入门与进阶 Agent、多智能体、语音、RAG 和生成式界面等类别，可直接选择子项目运行，再替换模型或业务逻辑。

适合谁：想快速验证 AI 应用想法，或寻找 Agent、RAG 与多智能体参考实现的开发者
使用前准备：入门 Agent 通常需要对应模型的 API Key；克隆示例后需安装其 requirements.txt。

许可证：Apache-2.0

02｜orca
把多个终端编程 Agent 放进隔离工作区，并行执行、比较和审查结果

它能做什么：

- 并排运行不同 CLI 编程 Agent
- 分发同一提示并比较各自结果
- 从手机接收完成通知并追加指令
- 点击页面元素把界面上下文交给 Agent
- 批注代码差异并完成编辑与提交

核心亮点：隔离工作树并行试方案。每个 Agent 在独立 Git worktree 中执行同一任务，结果可集中比较并合并选中的方案，避免多个 Agent 同时修改一份工作区。

适合谁：同时使用多个 CLI 编程 Agent，需要并行试方案、集中审查与远程跟进任务的开发者
使用前准备：需安装桌面端；手机伴侣需与桌面应用配对，所用 Agent 必须能够在终端中运行。

使用提醒：手机端用于监控和引导桌面端 Agent，不能脱离桌面应用单独运行工作流。

许可证：MIT

为什么本期上榜：过去 7 天净增 +5,508 Star（历史快照核验）
GitHub 搜索：stablyai/orca

03｜OfficeCLI
让脚本和 AI Agent 在无 Office 环境中创建、读取与修改办公文档

它能做什么：

- 创建并编辑三类 Office 文档
- 查询文本、样式、公式与结构数据
- 渲染文档为 HTML 或页面截图
- 合并模板占位符生成批量文档
- 校验结构与排版问题后继续修正

核心亮点：让 Agent 看见文档成品。内置渲染引擎把文档转成 HTML 或 PNG，并配合实时预览形成“渲染、查看、修正”闭环，让 Agent 不只依赖文档结构猜测版面。

适合谁：需要在命令行、容器或 Agent 流程中批量生成、校验和改写办公文件的开发者与团队
使用前准备：直接使用只需自包含二进制，无需 Office 或 .NET 运行时；源码编译需 .NET 10 SDK。

使用提醒：驻留模式会延迟写入磁盘；交给其他程序读取前，需执行 save 或 close 刷新文件。

许可证：Apache-2.0

为什么本期上榜：过去 7 天净增 +4,290 Star（历史快照核验）
GitHub 搜索：iOfficeAI/OfficeCLI

04｜graphify
把代码、文档和媒体整理成可查询的知识图谱，减少逐文件搜索

它能做什么：

- 扫描项目并生成可点击的关系图
- 查询问题对应的局部子图
- 追踪两个概念之间的连接路径
- 解释单个节点及其关联来源
- 增量更新变更文件并重建图谱

核心亮点：用图谱代替逐文件搜索。代码通过 tree-sitter 在本地解析成节点和关系，再按社区聚类；查询时读取 graph.json 的相关子图，并区分原文提取与推断关系。

适合谁：需要理解陌生代码库、追踪模块关系，或让编程 Agent 先查项目全局结构的开发者
使用前准备：需要 Python 3.10+；推荐使用 uv 安装，也可使用 pipx。

使用提醒：代码解析可完全本地运行；文档、PDF 和图片的语义提取会调用助手模型或已配置的后端。

许可证：MIT

为什么本期上榜：GitHub Trending 显示过去 7 天 +8,611 Star
GitHub 搜索：Graphify-Labs/graphify

05｜superpowers
让编程 Agent 先澄清需求、确认设计，再按测试与审查流程完成开发

它能做什么：

- 梳理需求并分段提交设计确认
- 建立隔离工作树并检查测试基线
- 拆分包含路径与验证步骤的计划
- 执行测试驱动的红绿重构循环
- 审查实现并给出合并或保留选项

核心亮点：先确认设计，再让 Agent 开工。技能会在编码前追问目标并形成设计，经用户确认后生成任务计划，再由子 Agent 分项实现、检查规格与代码质量，最后验证测试结果。

适合谁：希望减少 Agent 直接开写和偏离需求，让研发过程保留设计确认、测试与代码审查的开发者
使用前准备：需按所用编程 Agent 的安装方式单独安装；同时使用多个 Agent 时需要分别配置。

使用提醒：设计需要用户确认后才进入计划，执行也需明确表示继续；不同宿主的安装与更新方式不同。

许可证：MIT

为什么本期上榜：过去 7 天净增 +4,585 Star（历史快照核验）
GitHub 搜索：obra/superpowers

06｜Vibe-Trading
把自然语言交易问题转成市场研究、策略回测与可检查的报告产物

它能做什么：

- 提问并生成带数据依据的市场研究
- 生成策略代码并运行跨市场回测
- 分析交易日志中的行为与偏差
- 组织多 Agent 投研与风险评审
- 导出报告及多平台策略文件

核心亮点：从问题走到可复查回测。流程先选择金融技能和数据源，再生成并执行策略，补充基准比较、验证与运行记录，最后交付报告、代码和可继续研究的会话产物。

适合谁：需要研究股票、加密资产或期货策略，并希望把问题、数据、回测和报告串成流程的研究者
使用前准备：需要受支持的 LLM API Key，或使用本地 Ollama；本地安装路径要求 Python 3.11+。

使用提醒：定位以研究、模拟和回测为主；自动交易仅在用户授权的经纪商与限制内启用，并可随时停止。

许可证：MIT

GitHub 搜索：HKUDS/Vibe-Trading

07｜hallmark
给编程 Agent 一套网页设计规则，生成、审查或重做更有区分度的界面

它能做什么：

- 根据需求从头构建网页界面
- 审查代码并输出问题清单
- 保留文案与品牌重新设计结构
- 提取参考设计的结构与视觉语言
- 输出可交接给其他工具的设计文档

核心亮点：先定结构，再过反模板检查。技能先按需求选择宏观结构和主题，再用规则门槛与输出前自检筛掉常见模板痕迹；不同需求会采用不同页面骨架，而非只换配色。

适合谁：使用 Claude Code、Cursor 或 Codex 制作网页，希望减少模板感并审查现有界面的设计开发者
使用前准备：可用 npx skills 安装，或把 SKILL.md 与 references 复制到对应 Agent 的技能目录。

使用提醒：study 模式拒绝像素级复刻和付费模板；audit 只输出问题清单，不会直接修改代码。

许可证：MIT
