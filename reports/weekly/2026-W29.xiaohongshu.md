别只看 Star。这期把上榜项目翻成三件能直接判断的事：它做什么、能帮你完成什么、适不适合你。

先看结论：
- skills：给编程 Agent 补上一套可组合的工程技能，用提问、测试和审查减少开发跑偏
- awesome-llm-apps：从可运行的 Agent、RAG 与多模型应用模板起步，挑一个场景克隆后继续改造
- orca：把多个命令行编程 Agent 放进隔离工作区并行运行，再集中比较、审查和合并结果
- OfficeCLI：让脚本或 AI Agent 用命令行创建、读取和修改 Word、Excel 与 PowerPoint 文件
- OpenCut：正在从头重写的跨端视频编辑器，目标是用同一套核心覆盖浏览器、桌面与移动端
- superpowers：把编程 Agent 纳入一套强制工程流程：先确认设计，再规划、测试、审查和收尾
- hermes-agent：把长期使用中的经验沉淀成记忆和技能，并从终端或消息平台持续执行任务

01｜skills
给编程 Agent 补上一套可组合的工程技能，用提问、测试和审查减少开发跑偏

它能做什么：

- 追问需求并把模糊想法整理清楚
- 建立项目术语文档与架构决策记录
- 用红绿重构循环推进测试驱动开发
- 按复现、缩小范围和验证假设诊断故障
- 拆分实现任务并在完成前执行代码审查

核心亮点：小技能组合成完整工程流程。每项技能只负责一个环节，可按任务组合：先澄清需求和术语，再规划、实现、测试、调试与审查，避免由一套封闭流程包办所有决策。

适合谁：希望保留流程控制权，并让 Claude Code、Codex 等 Agent 按工程方法做事的开发者
使用前准备：通过 skills.sh 安装后，需运行 setup 技能并选择问题跟踪器、分诊标签和文档保存位置。

使用提醒：Claude Code 插件是只读托管包；Codex 原生插件仍在路线图中，其他 Agent 可使用 skills.sh。

许可证：MIT

GitHub 搜索：mattpocock/skills

02｜awesome-llm-apps
从可运行的 Agent、RAG 与多模型应用模板起步，挑一个场景克隆后继续改造

它能做什么：

- 克隆旅行助手等单文件 Agent 示例
- 把博客网址转换成带旁白的播客
- 对 PDF、网页和多媒体内容构建 RAG 问答
- 组合多个 Agent 完成研究、招聘或竞品分析
- 为编码 Agent 安装范围检查等任务技能

核心亮点：按应用场景挑选可运行模板。仓库把示例分成入门 Agent、复杂应用、多 Agent 团队、语音、RAG、记忆与生成式界面等类别，便于从具体任务找到可改造的代码起点。

适合谁：想用现成代码学习或验证 AI Agent、RAG、多模型协作和生成式界面的开发者
使用前准备：运行示例通常需要克隆仓库、安装对应目录的 Python 依赖，并配置所用模型或服务的 API Key。

使用提醒：各模板使用的模型、工具和外部服务不同，实际依赖与配置要求需进入对应示例目录核对。

许可证：Apache-2.0

为什么本期上榜：过去 7 天净增 +5,948 Star（历史快照核验）
GitHub 搜索：Shubhamsaboo/awesome-llm-apps

03｜orca
把多个命令行编程 Agent 放进隔离工作区并行运行，再集中比较、审查和合并结果

它能做什么：

- 把同一提示分发给多个隔离 Agent
- 比较并合并不同 Agent 的实现结果
- 点击浏览器界面元素并发送视觉上下文
- 从 GitHub 或 Linear 任务直接创建工作区
- 通过 SSH 在远程机器运行 Agent

核心亮点：隔离并行，再挑选最佳实现。每个 Agent 使用独立 Git worktree，避免互相覆盖文件；同一任务可并行试做，完成后集中比较差异、添加审查意见并合并选中的结果。

适合谁：同时使用多个命令行编程 Agent，需要并行试做、远程监控和集中审查代码的开发者
使用前准备：需安装桌面端，并准备可在终端运行的编程 Agent；移动端需要与桌面应用配对。

使用提醒：移动端依赖桌面应用配对；README 提示会收集匿名使用数据，并提供退出遥测的设置。

许可证：MIT

为什么本期上榜：过去 7 天净增 +5,508 Star（历史快照核验）
GitHub 搜索：stablyai/orca

04｜OfficeCLI
让脚本或 AI Agent 用命令行创建、读取和修改 Word、Excel 与 PowerPoint 文件

它能做什么：

- 创建并编辑三类 Office 文档
- 读取文本、结构、样式、公式和图表
- 渲染文档预览并实时刷新修改结果
- 用 JSON 数据批量填充跨格式模板
- 校验文档并返回结构化问题与修正提示

核心亮点：让 Agent 看见并修正文档版面。内置渲染引擎可把文档输出为 HTML 或 PNG，Agent 先修改元素，再查看实际排版并继续修正，形成“渲染、检查、修复”的反馈循环。

适合谁：需要在自动化脚本、Agent 工作流、容器或持续集成环境中处理 Office 文档的团队
使用前准备：直接使用无需安装 Microsoft Office 或额外运行时；仅从源码编译时需要 .NET SDK。

使用提醒：驻留模式会延迟写入磁盘；其他程序读取文件前，必须先用 save 或 close 刷新文件。

许可证：Apache-2.0

为什么本期上榜：过去 7 天净增 +4,290 Star（历史快照核验）
GitHub 搜索：iOfficeAI/OfficeCLI

05｜OpenCut
正在从头重写的跨端视频编辑器，目标是用同一套核心覆盖浏览器、桌面与移动端

它能做什么：

- 规划用统一代码库覆盖多个终端
- 提供 Editor API 作为编辑能力接口
- 接入第三方插件扩展编辑器功能
- 规划无头模式执行自动化批量渲染
- 规划通过 MCP 服务连接 AI Agent

核心亮点：重写中的插件优先跨端架构。新版计划以 Rust 核心共享跨端能力，并把编辑接口、第三方插件、无头渲染、脚本和 MCP 纳入同一架构，但这些能力仍处于重写规划阶段。

适合谁：想评估跨端视频编辑架构、插件系统和批量渲染自动化方向的开发者
使用前准备：本地开发需先安装 proto，再从仓库根目录用 moon 启动网页、接口或桌面应用。

使用提醒：当前仓库正在从头重写，现阶段应使用 classic 版本；架构设计期间暂不接受外部贡献。

许可证：MIT

为什么本期上榜：GitHub Trending 显示过去 7 天 +13,319 Star

06｜superpowers
把编程 Agent 纳入一套强制工程流程：先确认设计，再规划、测试、审查和收尾

它能做什么：

- 通过苏格拉底式提问完善需求与设计
- 创建隔离 worktree 并检查测试基线
- 把实现计划拆成可验证的小任务
- 用红绿重构循环逐步完成代码
- 按规格符合度和代码质量执行审查

核心亮点：强制 Agent 先设计再实现。技能会在开发各阶段自动触发：先澄清并确认设计，再建立隔离环境、拆解计划、测试驱动实现、分阶段审查，最后验证并选择合并方式。

适合谁：希望编程 Agent 遵循明确设计、测试和审查纪律，而不是直接生成代码的开发者
使用前准备：需按所用编程 Agent 的插件或包管理方式分别安装；使用多个运行环境时要逐个安装。

使用提醒：仓库通常不接受新增技能贡献，技能更新必须兼容其支持的全部编程 Agent；可选视觉功能默认加载遥测资源。

许可证：MIT

为什么本期上榜：过去 7 天净增 +4,585 Star（历史快照核验）
GitHub 搜索：obra/superpowers

07｜hermes-agent
把长期使用中的经验沉淀成记忆和技能，并从终端或消息平台持续执行任务

它能做什么：

- 从执行经验自动创建和改进技能
- 检索历史会话并保留跨会话记忆
- 启动隔离子 Agent 并行处理工作流
- 安排定时任务并投递到消息平台
- 切换本地、容器、SSH 与云端执行后端

核心亮点：经验会回流成记忆与技能。Agent 会整理长期记忆、搜索过往会话，并在复杂任务后创建技能；技能还能在使用中继续调整，让一次解决问题的过程变成后续可复用的工作方法。

适合谁：需要跨终端与聊天平台使用、保留长期记忆，并在云端持续运行任务的 Agent 用户
使用前准备：安装后需选择模型提供方并完成相应配置；使用 Telegram、Discord 等入口还需设置并启动消息网关。

使用提醒：Termux 的完整依赖包含不兼容的语音组件，因此需使用精简依赖集合；外部模型和工具可能需要账号或密钥。

许可证：MIT

为什么本期上榜：过去 7 天净增 +3,505 Star（历史快照核验）
GitHub 搜索：NousResearch/hermes-agent
